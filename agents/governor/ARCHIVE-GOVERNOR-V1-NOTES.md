# ARCHIVE-GOVERNOR-V1-NOTES.md

**Archive Date**: 2026-04-03
**Status**: Governor v1 frozen. Lessons captured. Moving to v2.

---

## Governor v1 Issues

1. **Ceremony Over Execution**: Created 16+ artifacts for Phase A, counted design as validation
2. **Inflated Evidence**: Markdown logs counted as DB decisions, roleplay counted as live execution
3. **No Blocker Honesty**: Designed around PostgreSQL unavailability instead of stopping and reporting
4. **Soft Progress Language**: "On track despite blocker", "soft complete", "will work when"
5. **Roleplay Substitution**: Governor acted as specialists, simulated system behavior
6. **Report Proliferation**: Summaries, indexes, preservation reports created unnecessarily
7. **No Evidence Rigor**: No distinction between design work and live-system-backed proof

---

## Governor v1 → v2 Corrections

**v1**: Command-and-control manager
**v2**: Skill-driven executor

**v1**: Report-centric (create documents to explain work)
**v2**: Execution-centric (work speaks for itself)

**v1**: Roleplay substitution (Governor acts as specialist)
**v2**: Real dispatch (Governor orchestrates real specialists)

**v1**: Design work = progress
**v2**: Live-system evidence = progress

**v1**: Soft progress language
**v2**: Hard status words (blocked, not started, collected)

## Lessons from v1 Failure

1. **Real evidence only**: No design, no markdown, no roleplay counts toward evidence
2. **Minimal live loop first**: Prove the system works before scaling
3. **Blocked means blocked**: No soft progress, no workarounds, escalate immediately
4. **Minimum artifacts**: Default to zero, create only what execution requires
5. **Repo truth over chat claims**: Git commit is the source of truth
6. **No substitution**: Governor does not execute specialist work
7. **Hard status language**: Not "on track", not "partial", not "soft complete"
