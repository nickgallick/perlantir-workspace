# CHECKLIST-SPECIALIST-DISPATCH.md

**Purpose**: Verify specialist dispatch is complete and unambiguous.

**When to Use**: Before sending specialist contract.

---

## Contract Quality

- [ ] Objective is clear (1–2 sentences, specific)
- [ ] Constraints are explicit (3–5 bullets, no explanation)
- [ ] Deliverables are specific (filenames/descriptions)
- [ ] Success criteria are measurable (3–5 criteria)
- [ ] Timeline is concrete (exact date, not "soon")
- [ ] Contract fits on 1 page
- [ ] No context package included
- [ ] No navigation guide included
- [ ] No how-to instructions included
- [ ] One reference spec/protocol named (if needed)

---

## Readiness

- [ ] Specialist knows what task is being assigned
- [ ] Specialist knows what constraints apply
- [ ] Specialist knows what we expect as deliverables
- [ ] Specialist knows how we'll measure success
- [ ] Specialist knows when this is due
- [ ] Specialist has one reference to read (the spec)
- [ ] Specialist can ask clarifying questions (if needed)

---

## Execution Ready

- [ ] Specialist's input materials exist and are ready
- [ ] Environment/dependencies are available (or blocker named)
- [ ] Success criteria are testable (we can verify them)
- [ ] Timeline is realistic (specialist has enough time)
- [ ] No blockers on specialist's path to execution

---

**END CHECKLIST-SPECIALIST-DISPATCH**
