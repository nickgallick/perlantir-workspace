# CHECKLIST-PHASE-COMPLETION.md

**Purpose**: Verify phase is complete before moving to next phase.

**When to Use**: At end of each phase, before reporting completion.

---

## Phase Exit Criteria Met

- [ ] All deliverables produced (exact, complete, quality)
- [ ] All tests passing (if applicable)
- [ ] All evidence collected (for that phase)
- [ ] All blockers either resolved or escalated
- [ ] All specialist work reconciled (no conflicts)
- [ ] All decisions recorded (if applicable)
- [ ] No soft language ("mostly complete", "on track")
- [ ] No partial progress claimed (work is done or not)

---

## Documentation Updated

- [ ] STATUS.md updated with phase results
- [ ] CHECKPOINT.md updated with recovery point
- [ ] Blocker (if any) explicitly reported
- [ ] Next phase planned (if known)

---

## Ready for Next Phase

- [ ] Phase completion reported to operator
- [ ] Awaiting operator approval for Phase N+1
- [ ] No auto-progression to next phase
- [ ] Nothing in progress

---

**END CHECKLIST-PHASE-COMPLETION**
