# CHECKLIST-EVIDENCE-VALIDITY.md

**Purpose**: Verify evidence before claiming it. No inflated claims.

**When to Use**: Before updating evidence status. Before reporting progress.

---

## A-1: Specialist Task with Compiled Context

**Claim**: Specialist executed task with real compiled context

- [ ] Specialist received output from compile() call (not markdown package)
- [ ] compile() executed against live Nexus server (not simulated)
- [ ] Response included decision list, scores, formatted output
- [ ] Specialist received that response before starting task
- [ ] Specialist task mentions decisions from compile output
- [ ] Dispatch was real (not Governor acting as specialist)
- [ ] Evidence logged: timestamp + response + dispatch + task outcome

**If any box unchecked**: Do not claim A-1 evidence. Mark as NOT STARTED.

---

## A-2: Real Decision from Live Work

**Claim**: Decision was recorded in live Nexus system

- [ ] createDecision() API call executed against live Nexus
- [ ] Server responded with 200/201 + decision ID
- [ ] Response included all fields (title, created_at, tags, etc.)
- [ ] SELECT query confirms decision exists in decisions table
- [ ] Decision has UUID (auto-generated by server)
- [ ] Decision is queryable (SELECT returns the exact row)
- [ ] Evidence logged: API response + SQL query result

**If any box unchecked**: Do not claim A-2 evidence. Mark as NOT STARTED.

---

## A-3: Supersede Event Changes Compiled Context

**Claim**: Superseding a decision changes compile output

- [ ] Decision A was created and persisted (A-2 proof exists)
- [ ] Decision B was created and persisted (A-2 proof exists)
- [ ] Edge created: A → B, type="supersedes"
- [ ] A.status = "superseded" in database
- [ ] compile() called BEFORE supersede: response logged with scores
- [ ] compile() called AFTER supersede: response logged with scores
- [ ] Score for A differs: S_before ≠ S_after
- [ ] S_after ≈ S_before × 0.4 (supersede penalty)
- [ ] Before/after outputs logged as evidence

**If any box unchecked**: Do not claim A-3 evidence. Mark as BLOCKED or NOT STARTED.

---

## A-4: Operator Judgment

**Claim**: Operator has provided judgment on friction reduction

- [ ] Operator wrote explicit statement
- [ ] Statement includes: "reduced" OR "increased" OR "conditional on X"
- [ ] Statement is unambiguous (not implicit, not assumed)
- [ ] Statement is recorded in STATUS.md or project state
- [ ] Statement is attributed to operator (dated, sourced)

**If any box unchecked**: Do not claim A-4 evidence. Mark as NOT STARTED.

---

## Red Flags (Do Not Count)

❌ Design work (planning, specs, contracts)
❌ Markdown logs (not persisted in system)
❌ Simulated context packages
❌ Roleplay (Governor acting as specialist)
❌ Assumed/implicit approval
❌ "Will work when" language
❌ Soft progress ("50% complete", "on track")
