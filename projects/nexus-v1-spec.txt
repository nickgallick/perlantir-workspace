WHAT YOU’RE BUILDING  A TypeScript library + lightweight server that gives multi-agent systems structured project  memory with zero handoff loss. Three components:  1 .   Decision Graph   — directed graph of project decisions with typed relationships  (supersedes, requires, informs, blocks, contradicts)  2 .   Context Compiler   — assembles the RIGHT context for the RIGHT agent for the RIGHT  task, within a token budget. This is the core innovation.  3 .   Change Propagator   — when a decision changes, affected agents learn automatically  with role-appropriate notifications  The one-liner:   “Your agents talk. Nexus makes them think together.”  The technical claim:   “Vector search finds SIMILAR text. Nexus finds RELEVANT context —  based on agent role, task requirements, and a graph of causally connected decisions.”  What makes this different from Letta/Mem0/Zep/LangGraph memory:  Those store memory for SINGLE agents. Nexus coordinates memory across MULTIPLE  specialized agents.  Those use vector similarity search. Nexus uses role-aware relevance scoring + graph  traversal.  Those treat all stored information equally. Nexus understands that different agents need  different context from the same project.  The killer demo: same project, two agents get completely different context packages  based on their role and current task.  v1 scope:  Core library (decision graph + context compiler + change propagator + temporal)  REST API server
